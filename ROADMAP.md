# Roadmap

This document tracks the current public-shell direction and the next planned versions for Stage Overlay.

## Current Beta

Current focus:

- readable overlay previews and exports,
- simple local match-file import,
- portable Windows release prep,
- clearer first-run guidance and output controls.

Current constraints:

- Windows-first release target
- session-based match data only
- no installer yet
- no persistent match library yet

## Next Version

Priority improvements for the next release cycle:

- continue refining overlay layout, spacing, and typography
- improve imported match validation and user-facing error clarity
- tighten shooter resolution and stage/division data handling
- improve preview usability and export clarity
- expand preset and custom output-size workflows
- continue polishing vertical-friendly app layout behavior

## Planned After That

Medium-term improvements:

- persistent saved-match reopen flows
- packaged release polish for broader tester distribution
- stronger test coverage around parsing, preview generation, and export flows
- better preview inspection controls
- richer style controls and layout presets
- cleaner onboarding for first-time users

## Future Distribution Direction

Later release goals:

- move from portable beta distribution to an installer-based Windows release
- improve first-run experience for non-technical users
- harden release/version packaging discipline

## Longer-Term Product Improvements

Potential future versions may include:

- broader match import and reopen workflows
- more themes and layout presets
- improved onboarding and in-app guidance
- stronger import-provider abstraction for different match-data sources
